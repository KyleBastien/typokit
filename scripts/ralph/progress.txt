## Codebase Patterns
- Use `rstest run --passWithNoTests` for test commands (rstest 0.0.1 hangs without `run` subcommand)
- Import test functions from `@rstest/core` (not `rstest`): `import { describe, it, expect } from "@rstest/core"`
- Package tsconfigs extend `../../tsconfig.base.json` â€” don't put `paths` in base (causes TS5090). Paths live in root `tsconfig.json` with `baseUrl`.
- All packages follow same structure: `packages/<name>/package.json`, `tsconfig.json`, `src/index.ts`
- Nx targets defined in `nx.json` targetDefaults â€” individual package.json scripts match
- ESLint 9 flat config at root (`eslint.config.mjs`), packages use `eslint src/`
- PowerShell: use `ConvertTo-Json`/`ConvertFrom-Json` for JSON manipulation, not `python -c` with f-strings (escaping issues)
- dist/ and *.tsbuildinfo are gitignored â€” only commit src/ files
- Cross-package deps: add `"workspace:*"` to package.json dependencies + `"references": [{ "path": "../<dep>" }]` to tsconfig.json
- Error subclasses need `Object.setPrototypeOf(this, new.target.prototype)` for correct instanceof checks
- ESLint enforces `import type` for type-only imports â€” separate value imports from type imports
- No `@types/node` in core â€” use `globalThis` cast for `process.env` access, not bare `process`
- Duck-type check for third-party errors (e.g. Typia TypeGuardError) by `error.name` + property shape, not imports
- Rust crate in transform-native: `Cargo.toml` + `build.rs` at package root, `.rs` files in `src/` alongside `.ts` files
- napi-rs v3 + swc_ecma_ast v21 uses `Wtf8Atom` for string literals â€” use `.to_string_lossy()` not `Display`/`format!`
- For native addon in ESM: use `createRequire` from `"module"` with an `env.d.ts` shim declaring `ImportMeta.url`
- `target/` and `*.node` are gitignored â€” Rust build artifacts and native addons not committed
- `napi build --platform --dts src/index.d.ts` auto-regenerates d.ts â€” keep NativeBindings interface in index.ts in sync with lib.rs napi exports
- BTreeMap for deterministic JSON serialization in Rust; serde `skip_serializing_if` for clean optional fields
- When adding napi exports for complex types, define separate `#[napi(object)]` structs prefixed with `Js` â€” convert to/from internal types in lib.rs
- `napi build --platform --dts src/index.d.ts` regenerates both `index.d.ts` and `index.js` â€” don't commit `index.js`
- `--js false` napi flag generates a file named `false` â€” add both `index.js` and `false` to gitignore for transform-native
- `buildPipeline()` writes to `.typokit/` directory â€” use `regenerated` flag to check if cache was hit
- `GeneratedOutput` is `{ filePath, content, overwrite }` â€” BuildResult.outputs expects GeneratedOutput[], not string[]
- No `@types/node` pattern: create `env.d.ts` with module declarations, use dynamic imports with inline type casts
