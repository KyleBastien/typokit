## Codebase Patterns
- Benchmarks package uses Nx auto-infer (no project.json needed) — consistent with all packages
- pnpm-workspace.yaml `packages/*` glob auto-detects new packages
- Raw baseline apps use hand-written routing (no framework) — same 5 endpoints as TypoKit apps
- ESLint enforces LF line endings — use `eslint --fix` to auto-convert CRLF on Windows
- ESLint forbids `import()` type annotations — use explicit `import type` at top of file
- Server adapter apps follow shared-routes pattern per platform (shared-routes.ts, shared-routes-bun.ts, shared-routes-deno.ts)
- Root .gitignore has `target/` for Rust and `.typokit/` for generated files
- Axum 0.8 path params use `{id}` syntax (not `:id`)
- For rusqlite in Axum, use `Arc<Mutex<Connection>>` + `spawn_blocking` since Connection is Send but not Sync
- Build target (tsc -p tsconfig.lib.json) fails on Bun/Deno files; typecheck (tsc --noEmit) is the real quality gate
- Competitor apps use each framework's recommended validation (JSON Schema for Fastify, TypeBox for Elysia, manual for others)
