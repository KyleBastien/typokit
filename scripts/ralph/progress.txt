## Codebase Patterns
- Existing generators (openapi_generator.rs, test_stub_generator.rs) use string-based code generation, returning String directly
- Handler entity files (src/handlers/{entity}.rs) use overwrite: false; mod.rs files use overwrite: true
- Entity name resolution: strip "Routes" suffix from handler_ref, then try singular forms (strip 's', 'ies'→'y') to match type_map keys
- TypeMetadata from type_extractor.rs contains `jsdoc: Option<HashMap<String, String>>` for JSDoc tags like @table, @id, @generated, @format, @unique, etc.
- PropertyMetadata contains `type_str` (stringified TS type), `optional`, and `jsdoc` for per-property annotations
- HashMap keys in type_map are type names, values are TypeMetadata. Properties within are also HashMap-keyed by property name
- All generators sort output deterministically (BTreeMap or sorted Vec) for reproducible codegen
- cargo check and cargo test both work from packages/transform-native directory
- Tests use internal parser::parse_typescript() to create test data from TS source strings
- Type extractor only processes interfaces (TsInterfaceDecl), NOT type aliases — utility types (Omit/Partial/Pick) are not in type_map
- Union literal types in properties are stored as `"active" | "inactive"` (with quote chars around each variant)
- Use BTreeMap for collecting resolved types to ensure deterministic iteration order
- rust_codegen::generate() accepts both type_map AND routes — sub-modules should accept the inputs they need
- Handler refs follow "EntityRoutes#METHOD /path" format — derive entity module by stripping "Routes" suffix and snake_casing
- @table entities have jsdoc.contains_key("table"); the value is the SQL table name (fall back to snake_case + "s")
- @generated properties: value "uuid" → Uuid::new_v4(), value "now" → Utc::now(); these are excluded from user input in INSERT
- @id property found by jsdoc "id" key or by property name "id"; used as PRIMARY KEY in migrations
- @unique fields get UNIQUE constraint in migration SQL; use collect on jsdoc.contains_key("unique")
- TS→PostgreSQL type mapping: string→TEXT, number→DOUBLE PRECISION, boolean→BOOLEAN, Date→TIMESTAMPTZ, arrays→JSONB, @integer→BIGINT
