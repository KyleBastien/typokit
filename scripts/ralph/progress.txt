- **Learnings for future iterations:**
  - `MiddlewareChain` entries from `@typokit/types` have no `priority` field — only `MiddlewareEntry` from core does. The sort was always a no-op in server adapters.
  - Server-native wraps `MiddlewareFn` (takes full request) into `Middleware` handler (takes `MiddlewareInput` subset). Method/path aren't in `MiddlewareInput`, so pre-built wrappers need a shared `currentReq` reference set per-request.
  - Other server adapters (hono, fastify, express) have the same middleware wrapping pattern and would benefit from the same optimization.
  - Lint (`eslint .`) fails on baseline due to CRLF line endings on Windows — not a regression.
